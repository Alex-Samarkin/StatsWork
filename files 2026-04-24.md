## Исходный код

### `src/StatsWorkbench.jl`

Главный модуль пакета `StatsWorkbench`.

Что делает:
- подключает зависимости Julia
- включает файлы из `Core`
- экспортирует ключевой внешний API:
  - `Workbook`
  - `open_workbook`
  - `close_workbook!`
  - `add_dataset!`
  - `vars`
  - `analyze`
  - `plot_report`
  - `text_report`
  - `save_report`

Важно: модуль также пытается подключить отсутствующие на данный момент файлы:
- `Data/data_space.jl`
- `Analysis/Analysis.jl`
- `Viz/Viz.jl`
- `Reports/Reports.jl`

Из-за этого пакет в текущем виде, вероятно, не загрузится без ошибок.

### `src/Core/workbook.jl`

Содержит базовую реализацию структуры `Workbook`.

Назначение:
- хранить имя workbook
- хранить пространство переменных (`VariableSpace`)
- хранить список результатов анализа
- хранить журнал сообщений
- хранить признак закрытия workbook

Реализованы функции:
- `open_workbook(name="Workbook")`
- `add_dataset!(wb, name, df)`
- `vars(wb)`
- `close_workbook!(wb)`

Это минимальный каркас для дальнейшего расширения.

### `src/Core/variables.jl`

Содержит структуру `VariableSpace` и логику регистрации датасетов.

Что реализовано:
- хранение набора датасетов
- отображение переменной на пару `(dataset, column)`
- автоматическая регистрация колонок датафрейма как символов
- получение переменной через `getvar`
- вывод списка доступных переменных через `vars`

Это ядро идеи “единого пространства переменных”.

### `src/Core/results.jl`

Содержит абстрактный тип результата анализа и конкретную структуру `AnalysisResult`.

Структура результата включает:
- идентификатор
- группу анализа
- тип анализа
- заголовок
- произвольные данные
- таблицы
- метаданные

Это хороший базовый контейнер для будущих статистических процедур.
